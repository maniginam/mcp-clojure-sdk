Key Design Decisions

  • Bidirectional lsp4clj: Both client and server use ChanServer — same endpoint type, different multimethod handlers
  • Dynamic vars for handler context: *request-meta*, *server*, *context* avoid threading server references through every handler
  • Response coercion: Handlers can return strings/nil/maps/sequences — SDK wraps them correctly
  • Capability tracking: Registration/deregistration auto-updates :capabilities atom — no stale capabilities advertised
  • Underscore preservation: Custom JSON key transformer preserves _meta (MCP protocol field) through serialization
